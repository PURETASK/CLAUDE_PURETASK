# Task Intake

Purpose: keep a lightweight kickoff record for each implementation task so every change follows project playbooks.

## How to use
- Copy `docs/playbooks/feature-kickoff-template.md`.
- Save the copy in this folder with a date and short task slug.
- Complete the kickoff before coding.
- Reference the kickoff in PR notes.

## Filename pattern
- `YYYY-MM-DD-<short-task-name>.md`
- Example: `2026-05-06-phase-2-auth-hardening.md`
