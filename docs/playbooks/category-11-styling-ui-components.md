# Category 11 Playbook: Styling and UI Components

## Build Rules
- Use Tailwind CSS utilities.
- Use shadcn/ui primitives as base UI.
- Use `lucide-react` for icons.
- Prefer semantic theme tokens/CSS variables over raw hex values.
- Use Tailwind spacing scale for consistency.

## Never Do
- Do not hardcode one-off colors without reason.
- Do not create duplicate primitives when shadcn already provides one.

## Done Checklist
- UI changes align with token and spacing conventions.
- Reusable UI is abstracted appropriately.
- Visual behavior is consistent with existing app patterns.
