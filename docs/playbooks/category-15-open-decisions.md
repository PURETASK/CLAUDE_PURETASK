# Category 15 Playbook: Open Decisions

Purpose: track intentionally deferred choices so they are made at the right time.

## Open Decision Register
- Email provider: decide in Phase 4.
- Push notifications provider: decide in Phase 6.
- SMS provider: decide in Phase 6.
- Analytics provider: decide in Phase 10.
- Support chat provider: decide in Phase 10.
- Image CDN strategy: decide in Phase 6.
- Background queue platform: decide in Phase 7.

## Decision Process
- Decide only when phase context requires it.
- Capture selected option and rationale.
- Update:
  - `docs/puretask-decisions.md`
  - `docs/puretask-build-playbook.md` (if behavior changes)
  - `AGENTS.md` (if coding convention changes)

## Done Checklist
- Deferred decisions are not prematurely implemented.
- Any newly decided item is documented with date + reason.
