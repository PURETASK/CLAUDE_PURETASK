# Cleaner Onboarding Policy (Phase 4)

Status: Draft for implementation use.

## Launch Coverage
- Launch state coverage: California-first.
- Expansion to additional states requires legal and disclosure updates.

## Background Check Handling
- `clear`: eligible for standard admin approval review.
- `consider`: always route to admin review (never auto-reject).
- `failed`: route to admin review with explicit reasoning and adverse action workflow.

## Appeals and Adverse Action
- Rejection based on background report must follow FCRA adverse action flow.
- Initial support contact for appeals: `support@puretask.co` (update if different).
- Keep an auditable log of admin decisions and notices.

## Approval Preconditions
Admin can approve only when all are complete:
1. Identity verification = verified
2. Background check = clear (or reviewed consider path)
3. Stripe Connect onboarding completed
4. Photo etiquette training completed
5. Tax info captured

## Notes
- This policy doc is an implementation reference and must be reviewed by counsel before production launch.
